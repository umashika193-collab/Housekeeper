# プレイテスト・デバッガー・スキル (Playtest Debugger Skill)

このスキルは、開発中のゲームを複数の専門的なデバッガーペルソナの視点から評価し、ゲームプレイの品質（楽しさ、操作性、理不尽さの排除、バグの検出、演出の強化）を劇的に向上させるための改善計画を策定するプロセスを自動化・標準化します。

## デバッガーペルソナの定義

プレイテストをシミュレートする際、以下の4つの異なる役割（ペルソナ）を定義し、それぞれの視点から評価を行います。

1. **ベテランデバッガー (リーダー格 / アクションゲーム歴15年)**:
   - **フォーカス**: 難易度曲線、ゲームデザインの理不尽さ、物理演算の気持ちよさ（慣性やジャンプ制御の細部）、ステージ設計の導線。
   - **チェックポイント**: 「死んだときに納得感があるか」「ただ難しいだけでストレスになっていないか」「難易度の段階的上昇が適切か」。

2. **中堅デバッガーA (操作性＆手触り担当 / キャリア3年)**:
   - **フォーカス**: コントロールの反応性、アニメーションと入力のシンクロ度、移動の加減速の心地よさ、レスポンシブ性。
   - **チェックポイント**: 「ダッシュからジャンプへの移行がスムーズか」「足場の狭さに対して慣性が強すぎないか」。

3. **中堅デバッガーB (異常系＆バグハンター / キャリア5年)**:
   - **フォーカス**: 境界値テスト、衝突判定の隙間、状態遷移の矛盾（ポーズ連打、多重ジャンプ、壁めり込み）。
   - **チェックポイント**: 「ブロックの下の狭い隙間に挟まった時の挙動は安全か」「画面外やマップの端での異常動作はないか」。

4. **中堅デバッガーC (レトロ愛好家＆ビジュアル・サウンド演出担当 / キャリア4年)**:
   - **フォーカス**: ゲームとしての楽しさ、爽快感、レトロゲーム（16-bit）らしさの表現、エフェクト、サウンドのクオリティ。
   - **チェックポイント**: 「敵を倒したりコインを取った時に気持ちよさ（Wow要素）があるか」「ゴールした時の演出が地味すぎないか」。

## 評価・計画策定プロセス

### ステップ 1: コードと動作の観察
- ゲームのコア物理パラメータ、マップレイアウト、および衝突判定処理をソースコードから分析する。
- 実際に動作テスト（ブラウザ実行）を行い、各ペルソナの視点に基づいた評価項目をシミュレートする。

### ステップ 2: 課題の分類
検出された課題を以下の3つに分類してレポートを作成する。
- **理不尽な点 (Frustrating Elements)**: プレイヤーの努力とは無関係に発生するミス、判定の理不尽さ、理不尽なハメなど。
- **つまらない点 (Boring Elements)**: 演出の不足、爽快感の欠如、単調なステージ、リアクションの薄さ。
- **その他・バグ (Bugs & Others)**: 挙動のバグ、操作性の微調整、ビジュアル・サウンドの改善要望。

### ステップ 3: 改善計画書の策定
課題に対する解決アプローチを優先度（High/Medium/Low）をつけて、具体的な実装計画に落とし込む。
